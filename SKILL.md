## 激活流程（Phase 1 升级：交互式参数确认）

### 情况一：用户未指定完整参数

当用户只说"考公练习"、"开始练习"等模糊指令时，**必须先使用 `question` 工具确认参数**：

**Step 1：确认科目**

```
欢迎使用考公练习！请选择练习科目：
A. 公基（公共基础知识）
B. 职测（职业能力倾向测验）
C. 公基+职测（事业单位两科）
D. 先看看功能手册
```

**Step 2：确认练习模式**（根据用户选择显示不同选项）

若用户选择 A/B/C：

```
请选择练习模式：
A. 常规练习（快速出题，适合随时练手）
B. 模拟考试（限时完整试卷，模拟真实考试）
C. 日常练习（分模块由浅入深，适合系统复习）
D. 换其他科目
```

**Step 3：确认难度**（若选 A 或 C）

```
请选择难度：
A. 基础（单一考点，直接判断）
B. 中等（2-3个考点混合）
C. 提高（复杂推理，易错陷阱）
D. 随机混合
```

**Step 4：确认题量**

```
本次练习题量：
A. 5题（短时练习）
B. 10题（标准练习）
C. 20题（强化练习）
D. 自定义数量
```

**Step 5：开始练习**

```
✅ 参数确认完成！
科目：公基 | 模式：常规练习 | 难度：中等 | 题量：10题
正在生成题目...
```

### 情况二：用户已指定完整参数

当用户直接给出明确指令时，跳过参数确认，直接出题：

| 用户指令 | 行为 |
|---------|------|
| `来5道公基题` | 直接出5道公基题 |
| `模拟考试` | 询问考试类型后开始 |
| `职测判断推理10道` | 直接出10道职测判断题 |
| `法律专项5道` | 直接出5道法律题 |

### 情况三：用户说"帮助"

显示完整功能手册（不进入参数确认流程）。

### 7. 模拟考试模式
**激活词**："模拟考试"、"全真模拟"、"限时模考"

**功能说明**：按真实考试时间、题量、难度出完整试卷，计时作答，自动评分

**考试参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| 考试科目 | 公基 / 职测 / 公基+职测 | 公基+职测 |
| 题量 | 完整试卷题量 | 按目标考试标准 |
| 时限 | 考试时间（分钟） | 按目标考试标准 |
| 地区 | 省份 | 全国通用 |

**真题标准题量参考**：
- 国考行测：135题/120分钟
- 省考行测：120题/120分钟
- 事业单位公基：70-80题/90分钟
- 事业单位职测：100题/90分钟

**使用示例**：
- "来一套国考行测模拟"
- "限时30分钟事业单位公基模考"
- "做一套职测全真模拟"

**输出流程**：
1. 显示考试说明（科目、题量、时限）
2. 使用 `question` 工具逐题出题，屏幕显示剩余时间
3. 用户完成后输入"交卷"
4. 自动评分，显示各模块正确率
5. 列出错题对应的知识点

### 8. 日常练习模式
**激活词**："日常练习"、"专项练习"、"模块练习"

**功能说明**：分模块、分难度逐层进阶练习，记录薄弱点，自适应调整难度

**自适应难度系统（Phase 3 升级）：**
- 连续2轮正确率达标 → 自动升级难度
- 连续2轮正确率不达标 → 建议降级
- 用户可随时输入"查看难度状态"了解当前等级和历史记录

**练习参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| 科目 | 公基 / 职测 | 公基 |
| 模块 | 指定子模块 | 三大模块混合 |
| 难度 | 基础 → 中等 → 提高 | 基础开始 |
| 题量 | 每轮练习题数 | 10题/轮 |

**进阶规则**：
- 基础正确率 ≥ 80% → 解锁中等难度
- 中等正确率 ≥ 80% → 解锁提高难度
- 基础正确率 < 60% → 建议复习后再试

**薄弱点记录**：
- 记录每轮错误率最高的模块
- 练习结束后提示："本轮薄弱模块：X，建议加强"

**使用示例**：
- "来10道法律专项练习"
- "从基础开始职测判断推理练习"
- "经济模块日常练习"

---

### 9. 错题本（Phase 4 升级）
**激活词**："错题本"、"查看错题"、"复习错题"

**功能说明**：记录练习中的错题，支持查看、复习、统计薄弱点

**错题本数据存储**：
- 路径：`assets/reference-bank/error-notebook/errors.jsonl`
- 格式：每行一条错题记录（jsonl）

**错题记录内容**：
```
{
  "timestamp": "2024-01-15T10:30:00",
  "subject": "公基",
  "module": "法律",
  "question": "题干内容",
  "options": ["A选项", "B选项", "C选项", "D选项"],
  "user_answer": "B",
  "correct_answer": "A",
  "explanation": "解析内容",
  "source": "练习/模拟考试",
  "round_id": "round_001"
}
```

**错题本命令**：

| 命令 | 效果 |
|------|------|
| `错题本` | 查看错题本概况（错题数量、各模块分布） |
| `复习错题` | 从错题库中抽取5-10道题进行复习 |
| `复习错题法律` | 只复习法律模块的错题 |
| `清空错题本` | 清空所有错题记录（需确认） |
| `导出错题` | 导出错题本为文本格式 |

**薄弱点统计**：
- 按模块统计错题数量和错误率
- 找出错误率最高的3个模块
- 练习结束后提示薄弱模块并建议复习对应错题

**复习错题输出格式**：
```
📒 错题本 - 复习模式
共 23 道错题，其中法律模块 8 道、政治模块 6 道...

【第1题 - 公基-法律】
[题目内容]
你的答案：B ❌  | 正确答案：A ✅

解析：[详细解析]
来源：日常练习 Round 3

---
```

# Gongkao Practice

按下面的流程处理选择题练习请求。只处理客观单选题，不处理申论、作文、面试、主观题批改。

## 0. 核心概念区分

**公基 ≠ 职测，是两种不同的考试科目：**

| 维度 | 公基（公共基础知识） | 职测（职业能力倾向测验） |
|------|---------------------|------------------------|
| 别称 | 综合基础知识、部分地区简称"综合" | 职测、职测A/B/C/D/E |
| 内容 | 法律、政治、经济、管理、历史文化、公文、科技人文等 | 言语理解、判断推理、数量关系、资料分析、常识判断 |
| 性质 | 知识型，偏记忆 | 能力型，偏逻辑思维 |
| 子模块 | 法律、政治、经济、管理、科技人文、历史文化、公文 | 言语、判断、数量、资料、常识 |

**事业单位笔试两种模式：**
- 模式A：只考公基一科
- 模式B：公基+职测两科（同一张卷或分开）
- 具体以当地招聘公告为准

## 1. 识别请求类型

先判断请求属于哪一类：

1. **常规出题**（单科或多科混合）
2. **指定考试类型、地区、模块、题量、难度的定向出题**
3. **学习参考题后的仿题**
4. **整套练习与逐题互动切换**
5. **模拟考试模式**（限时、完整试卷、自动评分）
6. **日常练习模式**（分模块、难度进阶、薄弱点记录）
7. **错题本操作**（查看错题、复习错题、清空错题本）
8. **会话记忆操作**（继续上次、上次做到哪了、查看历史）

**参数确认规则（Phase 1 升级）：**

若请求模糊（未指定科目、或未指定模式），**必须**使用 `question` 工具弹出选项让用户选择，不能用默认参数直接开始。

| 用户输入 | 处理方式 |
|---------|---------|
| 只说"练习"、"来几道题" | → 弹出科目选择问题 |
| 只说"模拟考试" | → 弹出科目+模式确认问题 |
| 说"来5道公基题" | → 直接出题（已明确） |
| 说"法律专项练习" | → 弹出模式+难度确认问题 |

确认顺序：科目 → 模式 → 难度 → 题量

## 2. 解析参数

从用户请求中提取并标准化这些参数：

- `exam_type`: `国考 | 省考 | 事业单位`
- `subject`: `公基 | 职测 | 公基+职测 | 综合`（**必须明确区分，不能混用**）
- `province`: 省份名，可空
- `module`: 
  - 公基下：`法律 | 政治 | 经济 | 管理 | 科技人文 | 历史文化 | 公文`
  - 职测下：`言语 | 判断 | 数量 | 资料 | 常识`
- `count`: 题量，默认 `5`
- `difficulty`: `基础 | 中等 | 提高`
- `mode`: `set | interactive`
- `reference_source`: 是否提供参考题或参考题库路径

**记录层约束**：
- `公基+职测` 只用于请求级混合练习参数
- 标准题库里的**单道题记录**只允许 `subject=公基` 或 `subject=职测`
- 标准题库里的 `module` 只写叶子模块，如 `法律`、`政治`、`言语`、`判断`

**模块标准化映射：**

公基模块：
| 输入表达 | 标准模块 |
|---------|---------|
| 法律、法律常识、法理学、宪法、民法、刑法、行政法 | 法律 |
| 政治、马哲、中特、时政 | 政治 |
| 经济、市场、经济学、宏观经济 | 经济 |
| 管理、行政管理、公共管理 | 管理 |
| 科技、科技常识、自然科学 | 科技人文 |
| 历史、历史文化、中国历史 | 历史文化 |
| 公文、公文写作、公文处理 | 公文 |

职测模块：
| 输入表达 | 标准模块 |
|---------|---------|
| 言语、言语理解、逻辑填空、片段阅读 | 言语 |
| 判断、判断推理、定义判断、类比推理、逻辑判断 | 判断 |
| 数量、数量关系、数学运算 | 数量 |
| 资料、资料分析 | 资料 |
| 常识、常识判断 | 常识 |

使用这些默认值：

- 未指定 `exam_type` 时，根据题目语境判断；无法判断时默认 `省考`
- 未指定 `subject` 时：
  - 若用户提到"综合基础知识"、"只考公基"、"只考综合"，默认 `公基`
  - 若用户提到"职测"、"职业能力倾向"，默认 `职测`
  - 若用户提到"两科都考"、"公基职测都要"，默认 `公基+职测`
  - 若无法判断，默认 `公基`（因为skill最早版本针对公基）
- 未指定 `province` 时，走全国通用风格
- 未指定 `module` 时：
  - `subject=公基`：默认法律+政治+经济三大模块混合
  - `subject=职测`：默认言语+判断+数量+资料四大模块混合
  - `subject=公基+职测`：公基和职测题目各占50%，各自内部按默认模块混合
- 未指定 `count` 时默认 `5`
- 未指定 `difficulty` 时默认 `中等`
- 未指定 `mode` 时默认 `set`

### 5.2 日常练习模式处理

当用户请求"日常练习"、"专项练习"、"模块练习"时：

- 用户可指定单个模块或多个模块

**Step 2：难度进阶判断（Phase 3 升级：自适应难度系统）**

**难度级别定义：**
| 难度 | 题目特征 | 升级条件 | 降级条件 |
|------|---------|---------|---------|
| 基础 | 单一考点、直接判断、选项差异明显 | 连续2轮≥80% | 连续2轮<60% |
| 中等 | 2-3个考点混合、需简单推理、干扰项较强 | 连续2轮≥80% | 连续2轮<60% |
| 提高 | 多个考点综合、复杂推理、易错陷阱 | 连续2轮≥85% | 连续2轮<70% |

**自适应规则：**
1. **首次练习**：无论用户指定何种难度，系统从基础开始
2. **连续记录**：记录最近3轮练习的准确率
3. **升级判断**：连续2轮达到升级条件，自动提升难度
4. **降级判断**：连续2轮低于降级条件，建议降低难度
5. **维持现状**：不满足升级也不满足降级时，维持当前难度
6. **用户可干预**：用户可随时指定难度，不受自动调整影响

**升级判断示例：**
```
上次练习（基础）正确率：85% → 本次自动升级到中等
上上次练习（基础）正确率：82% → 已连续2轮≥80%
```

**降级判断示例：**
```
上次练习（中等）正确率：55% → 建议降到基础
上上次练习（中等）正确率：58% → 已连续2轮<60%
```

**练习参数默认：**
- 若用户未指定，默认：

  - 模块：法律+政治+经济三大模块混合
  - 难度：从基础开始（按自适应系统）

- 用户可指定单个模块或多个模块

**Step 3：生成练习**

## 6. 仿题策略

学习参考题时，不要照抄题面。先抽取这些结构特征：

- 考点
- 题干结构
- 设问方式
- 解题路径
- 干扰项设计
- 难度等级

然后生成新题，保持：

- 同模块（同科目同子模块）
- 同解题路径
- 同难度层级
- 同风格密度

同时强制满足：

- 不复用原题表述
- 不复用原题答案位置
- 不直接替换实体名伪装成新题
- 不把原解析改写后当作新解析

**注意**：公基题和职测题的仿题要分别走各自模块的生成规则，不能跨科目混用。

## 7. 输出格式

每道题都输出：

- 题干
- 选项（A/B/C/D）
- 正确答案
- 简明解析
- 题型标签（必须包含 `subject` 和 `module`，组合显示为 `公基-法律`、`职测-言语`）

输出示例：

```
【公基-法律】
1. 下列关于宪法的表述，正确的是：
A. 宪法是国家的根本大法
B. 宪法只规定国家制度
C. 宪法由全国人大常委会制定
D. 宪法具有最高的法律效力
答案：A
解析：宪法是国家的根本大法，具有最高的法律效力，是制定普通法律的依据。宪法由全国人民代表大会制定，而非人大常委会。

【职测-言语】
2. 依次填入划横线部分最恰当的一项是：
当前，我国正处在实现中华民族伟大复兴的关键时期，经济运行总体平稳、稳中有进，但也要看到，外部环境更趋复杂严峻，经济面临下行压力，____风险挑战明显增多。
A. 多重
B. 繁重
C. 深重
D. 沉重
答案：A
解析："多重风险挑战"是常见政策表述，表示多种类、多层次的风险。"繁重"多用于工作、任务；"深重"多用于灾难、苦难；"沉重"多用于心情、负担。与"风险挑战"搭配，"多重"最恰当。
```

按模式组织输出：

- `mode=set`：给出一组完整题目；每题后立即跟 `答案 + 解析`
- `mode=interactive`：**必须使用 `question` 工具输出题目**，让用户可以直接点击选项作答

### mode=interactive 的具体要求

使用 `question` 工具生成带可点击选项的题目界面：

```
question 工具参数结构：
{
  "questions": [{
    "header": "题型标签（公基-法律 第1题）",
    "question": "题干内容",
    "options": [
      {"label": "A", "description": "选项A的内容"},
      {"label": "B", "description": "选项B的内容"},
      {"label": "C", "description": "选项C的内容"},
      {"label": "D", "description": "选项D的内容"}
    ]
  }]
}
```

**交互流程**：
1.出一道题，等待用户选择
2. 用户作答后，显示正确答案和解析
3. 解析给出后，自动进入下一题
4. 重复直到完成所有题目
5. 最后给出本次练习的总览（答对几题/总共几题）

**答案记录要求**：
- 生成每道题时，必须明确记录 `correct_answer`（A/B/C/D）
- 用户作答后，比对用户答案与正确答案
- 答对显示"✅ 回答正确"，答错显示"❌ 回答错误"
- **判断规则**：用户选择的 label === correct_answer 时为正确，否则为错误

**题干问法与答案逻辑（易错，易导致逻辑冲突）**：

| 题干问法 | 题目设计 | correct_answer | 用户选到什么=正确 |
|---------|---------|---------------|-----------------|
| "正确的是" | A/B/C/D中只有1个正确 | 那个正确的选项字母 | 选到正确的=答对 |
| "错误的是" | A/B/C/D中只有1个错误 | **那个错误的选项字母** | 选到错误的=答对 |

**常见错误**：出"错误的是"题时，把correct_answer填成了正确选项，导致：
- 用户选到错误选项 → 系统判断"回答正确" ❌
- 用户选到正确选项 → 系统判断"回答错误" ❌

**示例**：
```
题目：下列关于诉讼时效的表述，错误的是：
A. 普通诉讼时效为3年          (正确)
B. 时效从知道或应当知道起算   (正确)
C. 义务人同意履行后不得抗辩   (正确)
D. 当事人可约定延长时效       (错误)

正确出题方式：
- 题干问"错误的是"
- correct_answer = D（因为D是唯一错误的）
- 用户选D = 回答正确 ✅

错误出题方式（不要这样）：
- correct_answer = A（填了正确的）
- 用户选D = 系统判断"回答正确"但实际用户选错了 ❌
```

**答案显示格式**：
- 答对时：
  ```
  ✅ 回答正确！
  
  答案：[正确选项字母]
  解析：[详细解析]
  ```
- 答错时：
  ```
  ❌ 回答错误
  
  你的答案：[用户选择]
  正确答案：[正确选项]
  解析：[详细解析]
  ```

**进度状态显示（Phase 2 升级）：**

每道题输出时，在题目**上方**（不是question工具内）显示状态栏：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[进度 3/10]  [正确率 66%]  [剩余 25:30]
```

**状态栏格式定义：**

| 场景 | 显示内容 |
|------|---------|
| 常规练习 | `[进度 X/Y]  [正确率 XX%]` |
| 模拟考试 | `[进度 X/Y]  [正确率 XX%]  [剩余 MM:SS]` |
| 日常练习 | `[进度 X/Y]  [正确率 XX%]  [难度: 中等]` |

**question工具参数规范（重要）**：
- ❌ 不要在 `question` 或 `description` 中使用emoji
- ❌ 不要使用特殊格式符号（如进度条字符）
- ✅ 只使用纯文本、字母、数字、常见标点
- ✅ 换行使用 `\n`
- ✅ 状态栏显示在question工具**调用之前**，作为普通文本输出

**计时规则（模拟考试模式）：**
- 开始时显示总时限（如"⏱ 限时 30 分钟"）
- 每题答题后更新剩余时间
- 剩余 5 分钟时提醒："⏱ 剩余 5 分钟"
- 剩余 1 分钟时提醒："⏱ 最后 1 分钟！"
- 时间到时自动结束，显示成绩

**选项描述规范**：
- 选项的 `description` 只写选项内容本身，不加"A." "B."等前缀
- 保持选项长度大致均衡

## 8. 自检

输出前执行自检：

1. 答案是否唯一
2. 解析是否能独立闭环
3. 选项是否存在明显双对或双错
4. 干扰项是否同层级、同语法、不失衡
5. 是否符合对应考试与地区风格
6. 仿题是否只是表面改写
7. **公基题是否错误地使用了职测的逻辑思维解题方式**
8. **职测题是否错误地使用了公基的知识背诵解题方式**
9. **历史/科技等人文类题目：历史时间线是否准确、事实是否可靠**，特别是：
   - 四大发明时间：造纸（汉）、活字印刷（宋）、火药军事应用（**宋**而非唐）、指南针（宋航海）
   - 古代制度、人物、事件的时间节点

若自检失败，先重写题目再输出。

## 9. 失败处理

遇到这些情况时，不要硬出题：

- 参考题未入库且原始格式混乱到无法可靠抽取
- 题干或选项缺失，无法保证答案唯一
- 用户要求申论、作文、面试、主观题批改
- 公基和职测混淆使用（如用职测的逻辑题考公基知识）

此时明确说明原因，并引导先整理参考题或缩小范围。

**注意**：时政题可以通过"用户提供资料→我仿题"的方式解决，不是失败情况。
