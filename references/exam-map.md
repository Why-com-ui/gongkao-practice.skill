# Exam Map

## 考试类型标准化

把用户表达标准化为以下三类：

| 输入表达 | 标准考试类型 |
| --- | --- |
| 国考、国家公务员考试、中央机关及其直属机构招录、公务员国考 | 国考 |
| 省考、联考、地方公务员考试、各省公务员考试 | 省考 |
| 事业单位、事业编、事业单位联考 | 事业单位 |

## 科目标准化

**公基和职测是两种不同科目，必须明确区分：**

| 输入表达 | 标准科目 |
| --- | --- |
| 公基、公共基础知识、综合基础知识、综合基础 | 公基 |
| 职测、职业能力倾向测验、职业倾向能力测验 | 职测 |
| 公基+职测、两科都考、都要、都要考 | 公基+职测 |

**注意**：
- "综合"在不同地区可能指公基或职测，**必须通过当地公告确认**
- 本skill默认"综合基础知识"归为公基
- `公基+职测` 只用于请求级组合练习；**单道题的标准记录里，`subject` 只能是 `公基` 或 `职测`**

## 省份处理规则

1. 若 `exam_type != 省考`，除非用户特别要求，不叠加省份差异。
2. 若 `exam_type = 省考` 且指定省份，先使用全国联考基线，再读取 [province-diffs.md](province-diffs.md)。
3. 若未指定省份，使用全国通用风格：
   - 题干清晰
   - 选项均衡
   - 难度不过度极端
   - 不引入强地方知识依赖

## 混合练习默认分配

### 公基单科

- 按法律、政治、经济、管理、科技人文、历史文化、公文平均分配
- 余数优先补给法律（因为法律题最稳定）
- 若用户指定模块，优先按指定出题

### 职测单科

- 按言语、判断、数量、资料平均分配
- 余数优先补给判断（因为判断题区分度最高）
- 若用户指定模块，优先按指定出题

### 公基+职测混合

- 公基题目占50%，职测题目占50%
- 各自按单科默认比例分配
- 输出时公基题在前，职测题在后

## 输出标签建议

每题都带上这些标签：

- `exam_type`
- `subject`（**必须标注**，如 `公基` 或 `职测`）
- `province` 或 `全国通用`
- `module`（叶子模块，如 `法律`、`言语`）
- 显示标签（由 `subject + module` 组合得到，如 `公基-法律` 或 `职测-言语`）
- `subtype`
- `difficulty`
