# Module Blueprints

按模块选择稳定、可复用、适合单选题生成的子题型。优先保证答案唯一和解析闭环。

**重要**：言语、判断、数量、资料属于**职测**（国考省考的行测也考这些）；公基是独立科目，规则不同。

---

## 职测模块

### 言语理解

**适用场景**：职测、国考省考行测

**推荐子题型**：

| 子题型 | 解题路径 | 干扰项套路 |
| --- | --- | --- |
| 逻辑填空 | 识别语境 -> 比较词义/搭配 -> 代入验证 | 近义词混淆、感情色彩错位、搭配不当 |
| 片段阅读 | 概括主旨 -> 排除细节/偷换 | 以偏概全、无中生有、因果倒置 |
| 语句表达 | 看衔接关系 -> 看话题一致 -> 看逻辑顺序 | 逻辑顺序错、照应缺失、重复啰嗦 |

**题干约束**：
- 题干保持完整语境，不要只给孤立词语
- 逻辑填空优先使用常见政策、治理、公共服务、经济民生语境
- 片段阅读避免模糊主旨或多中心表达
- **真实考试特征**：片段阅读通常100-150字，选项设置更精细

**解析模板**：
1. 点明语境核心
2. 比较正确项与干扰项差异
3. 给出代入后的顺畅性或逻辑性验证

### 判断推理

**适用场景**：职测、国考省考行测

**推荐子题型**：

| 子题型 | 解题路径 | 干扰项套路 |
| --- | --- | --- |
| 定义判断 | 提取定义要件 -> 比对选项 | 缺少关键要件、外延过宽、主体不符 |
| 类比推理 | 抽取词项关系 -> 比较关系一致性 | 表面相关、关系方向相反、层级错位 |
| 逻辑判断 | 识别论点论据 -> 找削弱/加强/前提 | 相关不等于因果、另有他因、偷换对象 |
| 事件排序 | 抓起点终点 -> 确认因果/时间顺序 | 中间步骤混淆、同阶段事件干扰 |

**题干约束**：
- 定义判断必须给出清晰定义
- 逻辑判断优先使用治理、管理、调查、政策执行等日常场景
- 不使用依赖图像识别的图形推理

**解析模板**：
1. 抽出题目要求判断的关系或要件
2. 说明正确项满足的核心条件
3. 指出至少两个干扰项的具体错误点

### 数量关系

**适用场景**：职测、国考省考行测

**推荐子题型**：

| 子题型 | 解题路径 | 干扰项套路 |
| --- | --- | --- |
| 工程问题 | 设总量为1 -> 列效率 -> 求未知 | 误把时间相加、误设总量、倒数关系错 |
| 行程问题 | 列路程=速度x时间 -> 分段计算 | 漏掉往返、相遇追及混淆 |
| 利润问题 | 成本售价折扣统一 -> 列方程 | 百分比基数混淆 |
| 排列组合/概率 | 分情况计数 -> 去重 | 重复计数、遗漏限制条件 |

**题干约束**：
- 数值设置要便于心算或简算
- 尽量让正确答案与干扰项之间有明确数量差异
- 不用过度刁钻的超长计算

**解析模板**：
1. 明确设元或设总量
2. 写出关键方程或等量关系
3. 代入选项或直接求解

### 资料分析

**适用场景**：职测、国考省考行测

**推荐子题型**：

| 子题型 | 解题路径 | 干扰项套路 |
| --- | --- | --- |
| 增长率 | 提取现期/基期 -> 列式 -> 估算 | 把增长量当增长率、基期用错 |
| 比重 | 提取部分/整体 -> 列式 -> 比较 | 分子分母调换、整体口径错 |
| 平均数 | 总量/数量 -> 估算 | 口径混淆、单位未统一 |
| 倍数与差值 | 列倍数或差额关系 | 把同比环比混淆、比较对象错 |

**题干约束**：
- 文本中直接给出数据，不依赖图像
- 口径、时间、单位要写清楚
- 优先做一步或两步计算题，避免超长材料

**解析模板**：
1. 说明提取了哪些数据
2. 写出公式
3. 解释估算或比较过程

### 常识判断

**适用场景**：职测（与公基有重叠，但侧重不同）

**推荐子题型**：

| 子题型 | 解题路径 | 干扰项套路 |
| --- | --- | --- |
| 法律常识 | 调用基础法律知识 -> 排除明显错误 | 主体混淆、条件缺失 |
| 政治常识 | 调用基本政治常识 -> 排除错误 | 表述绝对化、概念混淆 |
| 科技常识 | 调用基础科学知识 -> 排除不合理 | 违背常识、表述绝对 |
| 人文历史 | 调用基本人文常识 -> 排除不符 | 时间/人物错、违背常识 |

**注意**：职测的常识判断与公基有交集，但职测常识更侧重"判断"能力，公基更侧重知识储备。本skill的常识判断题可参考公基规则，但题目会更强调排除技巧。

### 图形推理

**适用场景**：职测判断模块中的图形推理部分

**ASCII可实现题型**：

| 子题型 | 解题路径 | 干扰项套路 |
| --- | --- | --- |
| 位置规律 | 观察平移、旋转、翻转规律 | 方向判断错误、周期遗漏 |
| 样式规律 | 识别加减同异、黑白运算 | 叠加规则混淆 |
| 数量规律 | 统计元素数量变化 | 计数遗漏、规律判断错误 |
| 简单组合 | 符号替换、规律延伸 | 变化趋势判断错误 |

**题干约束**：
- 使用ASCII字符组成图形，明确说明排列方式
- 九宫格格式：第1行/第2行/第3行
- 行列格式：第一列/第二列/第三列
- 选项用同等字符集表示

**解析模板**：
1. 观察图形特征，找出共同点或变化规律
2. 验证规律是否符合所有位置
3. 应用规律确定答案

**重要备注**：出题时必须附带说明，告知用户ASCII版本未覆盖对称性判断、空间重构等需要视觉判断的题型。

## 公基模块

公基详细范围见 [public-base.md](public-base.md)。

公基和职测的本质区别：
- **公基**：知识型，考查知道或不知道，侧重记忆和理解
- **职测**：能力型，考查会不会分析推理，侧重逻辑思维

### 推荐子题型

| 子题型 | 解题路径 | 干扰项套路 |
| --- | --- | --- |
| 法律常识 | 定位法律规则 -> 比较主体、条件、后果 | 绝对化表述、主体混淆、条件缺失 |
| 政治理论 | 比较概念内涵与表述边界 | 相近提法混淆、因果关系错 |
| 经济管理 | 判断概念定义与应用场景 | 概念偷换、范围张冠李戴 |
| 科技人文 | 调用稳定常识 -> 排除明显错误 | 时间敏感、冷门偏题、表述绝对化 |

### 公基约束

- 只做稳定知识点，不做强时政依赖题
- 优先使用法律、政治、经济、管理、科技人文的基础概念题
- 不使用需要精确记忆年份和最新文件号的题目
- **解析要直接点明知识点，不能只靠排除法**（与职测不同）

### 公基与职测常识的区分

| 维度 | 公基 | 职测常识 |
| --- | --- | --- |
| 解析方式 | 直接点明知识点 | 主要靠排除法 |
| 题目难度 | 知识深度较深 | 广而不深 |
| 备考重点 | 系统背诵 | 日常积累+技巧 |
